## Architecture

### Category System

**Key Innovation**: Custom hierarchical category system (not part of Hugo Stack theme default)

1. **Post Structure**: Every post has `categories: ["Primary", "Secondary"]` in front matter
2. **URL Structure**:
   - `/categories/` - All categories overview
   - `/categories/Primary/` - Primary category page
   - `/categories/Primary/Secondary/` - Secondary category page
3. **Auto-Generation**: `scripts/generate-categories.go` creates category pages before each build

**Critical Files**:
- `scripts/generate-categories.go` - Scans posts and generates category markdown files
- `layouts/categories/list.html` - Main categories index with collapsible UI
- `layouts/_default/category-primary.html` - Primary category template
- `layouts/_default/category-secondary.html` - Secondary category template

### How Category Generation Works

1. Go script scans `content/post/*/index.md` files
2. Extracts `categories: ["Primary", "Secondary"]` from front matter
3. Creates `content/categories/Primary/_index.md` with `layout: category-primary`
4. Creates `content/categories/Primary/Secondary.md` with `layout: category-secondary`
5. Hugo builds these pages using custom templates

**Important**: `content/categories/` is gitignored and regenerated on every build.

### Hugo Template Customization Rules

**Critical**: When customizing layouts, ONLY define the `{{ define "main" }}` block.

- ✅ DO: Define only `{{ define "main" }}...{{ end }}`
- ❌ DON'T: Add `{{ define "footer" }}` or `{{ define "sidebar" }}`
- ❌ DON'T: Call `{{ partial "footer/footer" }}` explicitly

The theme's `baseof.html` handles footer/sidebar automatically. Overriding them causes Disqus and other third-party script conflicts.

### Menu Configuration

Main menu items are defined in two ways:
1. **Global**: `config/_default/menu.toml` - Used for Categories
2. **Per-page**: Front matter `menu.main` section - Used for Archives, Links, Resume

Current menu order:
- Home (auto-generated by theme)
- Categories (weight: 2) - Defined in `menu.toml`
- Archives (weight: 3) - Defined in `content/page/archives/index.md`
- Links (weight: 4) - Defined in `content/page/links/index.md`
- Resume (weight: 5) - Defined in `content/page/resume/index.md`

**Note**: Items with same weight are sorted alphabetically.

**Requirements**:

- First element: Primary category
- Second element: Secondary category

## Troubleshooting

### Disqus Errors

**Symptom**: Disqus comments don't load on category pages
**Cause**: Custom layouts defining footer/sidebar blocks
**Fix**: Only define `{{ define "main" }}` block in custom layouts

### Menu Order Issues

**Symptom**: Menu items in wrong order
**Check**:
1. `config/_default/menu.toml` weight values
2. Page front matter weight values
3. Remember: Same weight = alphabetical sort

### Category Pages Not Generated

**Debug Steps**:
1. Check GitHub Actions logs for `go run scripts/generate-categories.go`
2. Verify Go version (^1.17.0 required)
3. Check if `content/categories/` directory exists after script runs
4. Verify posts have correct `categories: ["Primary", "Secondary"]` format

### Build Failures

**Check**:
1. Hugo version (uses latest extended)
2. Go version (^1.17.0+)
3. Template syntax errors (unmatched `{{ end }}` tags)
4. GitHub Actions workflow logs

## Layout Priority

Hugo resolves templates in this order:
```
layouts/categories/list.html              # Project override (highest priority)
    ↓ (if not found)
themes/.../layouts/categories/list.html   # Theme default
```

Always place custom layouts in project `layouts/` directory to override theme defaults.
